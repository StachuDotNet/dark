# Reflection layer (source material for WRAP-UP.md)

These four docs were written one-per-iter during the spike-end
reflection loop, then **consolidated into `../../WRAP-UP.md`** as a
single dense canonical doc.

Kept here because:
- The expanded form has more elaboration on individual points.
- Sections of WRAP-UP cite arguments developed at length here.
- Each doc stands alone if you want to read just that topic.

## Files

- `SPIKE-LEARNINGS.md` — what worked, what didn't, surprises, what
  we're still missing. The "what we built" retrospective.
- `INTEGRATION-PLAN.md` — 3-wave merge plan with per-wave LoC,
  test coverage, risks, and cross-cutting decisions to lock.
- `BIG-PICTURE.md` — horizon scan in 3 parts: things we haven't
  thought about, recursive-nature improvements, Dark-strength uses.
- `F-SHARP-TO-DARK.md` — 6-phase self-hosting roadmap (PDDMaterializer
  →`Stdlib.PDD.materialize`).

## Why archived

The user asked for **a single dense consolidated doc** at spike
wrap-up. WRAP-UP.md is that doc. These four were the inputs.

If you're editing the roadmap, edit `WRAP-UP.md`. These don't get
updated.
